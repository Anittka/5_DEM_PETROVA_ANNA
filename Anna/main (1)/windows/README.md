# Система учёта ассортимента и заказов детского магазина

Демонстрационная работа по специальности **09.02.07 «Информационные системы и программирование»**
Вариант В4

## Назначение проекта

Программное решение предназначено для ведения каталога товаров, обработки заказов и управления пользователями магазина детских товаров. Система поддерживает разграничение доступа в зависимости от роли пользователя.

## Используемые технологии

* Python 3.13
* SQLite
* CustomTkinter
* Pillow
* OpenPyXL

| Состояние товара    | Оформление           |
| ------------------- | -------------------- |

## Функции администратора

### Работа с товарами

* создание новых записей;
* изменение существующих товаров;
* прикрепление изображений;
* удаление товаров при отсутствии связанных заказов.

### Работа с заказами

* создание заказа;
* изменение данных заказа;
* удаление заказа;
* просмотр состава заказа.

### Импорт данных

Администратор может выполнять загрузку информации из Excel-файлов непосредственно через интерфейс программы.

## Формат импортируемых файлов

### products.xlsx

| Артикул | Наименование | Ед. изм. | Цена | Поставщик | Производитель | Категория | Скидка | Остаток | Описание | Фото |
| ------- | ------------ | -------- | ---- | --------- | ------------- | --------- | ------ | ------- | -------- | ---- |

### users.xlsx

| Роль | ФИО | Логин | Пароль |
| ---- | --- | ----- | ------ |

### orders.xlsx

| № заказа | Состав | Дата оформления | Дата получения | Адрес | Клиент | Код | Статус |
| -------- | ------ | --------------- | -------------- | ----- | ------ | --- | ------ |

Состав заказа хранится в формате:

```text
ART001,2,ART010,1,ART025,5
```

где после каждого артикула указывается количество единиц товара.

## Формирование исполняемого файла

```bash
pip install pyinstaller

pyinstaller --onefile --windowed ^
--name ShopManager ^
--add-data "windows;windows" ^
--add-data "data;data" ^
--add-data "images_source;images_source" ^
main.py
```

Скомпилированное приложение будет находиться в каталоге `dist`.

## Дополнительная информация

* База данных создаётся автоматически.
* Изображения копируются в рабочую директорию при импорте.
* Рекомендуемый размер файлов изображений — до 300×200 пикселей.
* При замене фотографии предыдущий файл удаляется автоматически.
* Для сохранения целостности данных запрещено удаление товаров, присутствующих в оформленных заказах.
